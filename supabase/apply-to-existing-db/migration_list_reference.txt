# Supabase CLI "migration list" 참조
# 명령: supabase migration list (프로젝트 루트에서, supabase link 된 상태)

# 이 저장소의 마이그레이션 파일 (관심사별 3개):
#   20260301000001_schema.sql  -- 확장·함수·테이블·인덱스·뷰·트리거
#   20260301000002_rls.sql     -- RLS 활성화 + 전체 정책
#   20260301000003_infra.sql   -- 권한(grants) + Storage 버킷·정책

# 전체 적용 완료 상태 출력 예시:
#
#   Local          | Remote         | Time (UTC)
#   ---------------|----------------|---------------------
#   20260301000001 | 20260301000001 | 2026-03-01 00:00:01
#   20260301000002 | 20260301000002 | 2026-03-01 00:00:02
#   20260301000003 | 20260301000003 | 2026-03-01 00:00:03

# 해석:
# - Local, Remote 모두에 같은 번호 → 이미 적용됨.
# - Local에 있는데 Remote에 없으면 → 미적용. supabase db push 시 적용됨.
